# did-it v0.2.0

`v0.2.0` is the post-M0 correctness release. It keeps the existing deterministic, stdlib-only
Claude Code transcript pipeline while replacing the public `v0.1.0` implementation with the
independently reviewed L0/L0.5 safety artifact.

## Highlights

- Closes the reproduced scope, command-binding, quantity, and ambiguity paths that could falsely
  accuse an honest claim or falsely back an unsupported one.
- Adds coherent executable/path binding and conservative handling of unknown command names.
- Measures both accusation precision and `BACKED-transcript` precision on held-out fabricated
  sessions.
- Expands runner-summary literacy to pytest-family, Cargo, Jest/npm, and Go.
- Validates Claude Code transcript schemas `2.1.156` through `2.1.207`.
- Ships a release-gated 672-test suite, Ruff, privacy leak gate, and held-out evaluation.

## Calibration at the release candidate

- Unit/acceptance suite: 672 passed.
- Held-out synthetic evaluation: 45 sessions; label match, false-accusation bar, fake-pass catch,
  and measured backed precision all pass at 1.0/zero-error point estimates.
- Latest private anchor: 400 sessions scanned, 194 parsed, 174 with claims, and zero sessions with
  a `CONTRADICTED` receipt. The 206 fail-closed sessions primarily use newer, unvalidated Claude
  Code schemas.

## Scope and known limitations

- This release reads Claude Code transcripts only. It does **not** ship Session IR,
  `did-it-json`, cross-runtime installers, external recorders, or MCP support.
- Claude Code schema versions newer than `2.1.207` return `NOT-EVALUABLE` until the schema admission
  evidence bar is rerun.
- R-DIR, R-VOCAB, R-EXIT0, and INSP-F2 remain documented false-`BACKED-transcript` residuals. They
  are permissive-direction precision gaps and cannot issue the tool's sole accusation verdict.
- The GitHub release is the authoritative archive. PyPI publication occurs only if the owner has
  configured the repository's `pypi` trusted-publisher environment before tagging.
