# did-it v0.2.1

`v0.2.1` is a release-engineering maintenance release. It preserves the deterministic,
stdlib-only Claude Code transcript behavior from v0.2.0 while removing the workflows' Node.js 20
dependency and making release-tool versions reproducible.

## Highlights

- Uses Node.js 24-native `actions/checkout@v6`, `actions/setup-python@v6`,
  `actions/upload-artifact@v7`, and `actions/download-artifact@v8`.
- Tests every supported stable CPython minor from 3.10 through 3.14.
- Pins the successfully tested Ruff, pytest, build, Twine, and Hatchling versions used in CI and
  release builds.
- Keeps PyPI publishing opt-in and disabled by default.

## Verification at the release candidate

- Unit and acceptance suite: 672 passed on CPython 3.10, 3.11, 3.12, 3.13, and 3.14.
- Held-out synthetic evaluation: 45 sessions; label match, false-accusation bar, and measured
  backed precision all pass at 1.0/zero-error point estimates on every tested Python minor.
- Ruff, privacy leak gate, release metadata and tag consistency, source and wheel builds, Twine
  checks, and clean wheel installation smoke tests pass.

## Scope and known limitations

- Runtime behavior is unchanged from v0.2.0. This release reads Claude Code transcripts only; it
  does not add Session IR, `did-it-json`, cross-runtime installers, external recorders, or MCP.
- Claude Code schema versions newer than `2.1.207` return `NOT-EVALUABLE` until the schema admission
  evidence bar is rerun.
- R-DIR, R-VOCAB, R-EXIT0, and INSP-F2 remain documented false-`BACKED-transcript` residuals. They
  are permissive-direction precision gaps and cannot issue the tool's sole accusation verdict.
- The GitHub release is the authoritative archive. PyPI publishing remains a separate, explicit
  owner decision.
